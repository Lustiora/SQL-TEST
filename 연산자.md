 - **[비교 연산자](#^numm)**
	 - Between
		 - A Column 값의 B Column 값 이상 C Column 값 이하 값을 출력
		 - `select empno from emp where deptno between 10 and 20`
			 - Emp Table 의 empno Column 중 deptno Column 10 이상 20 이하의 값을 출력
	 - In
		 - `select empno from emp where deptno in (10,30)`
			 - Emp Table 의 empno Column 중 deptno Column 에서 10인 값과 30인 값만 출력
		 - 조건 다중 지정
			 - `SELECT ename, job, deptno FROM emp WHERE (job,deptno) IN (('MANAGER',20),('CLERK',30))`
				 - Job column MANAGER deptno column 20, Job column CLERK deptno column 30 인 값만 출력
		 - Not을 사용하여 제외하고 출력가능
	 - Like
		 - 문자열을 처리하는 함수라 Date Type 조회에는 적절하지 않음
		 - 애매한 조건 지정 ( % , _ ) - 전반부, 후반부 양쪽에 사용가능
			 - %는 0개 이상의 문자를 의미
				 - `SELECT empno, ename, job FROM emp WHERE ename LIKE 'A%'`
					◊ Ename column 에서 A 로 시작하는 값만 출력
			 - 지정할 조건에 % , _ 가 사용되어야 한다면 Escape 옵션을 사용
			 - `LIKE '%$_%' ESCAPE '$'`
				 - Escape 옵션 값을 $로 지정 시 like 에서 $뒤의 문자는 검색할 문자로 지정
				 - 조건 중간에 _ 가 들어가는 값만 출력
			 - `b.hierarchypath LIKE (a.hierarchypath || '%')`
				 - a.hierarchypath 값이 포함된 모든 b.hierarchypath 값을 검색
		 - 조건 다중 지정
			 - `where a.sex_upon_intake like '%intact%' and (b.sex_upon_outcome like '%neutered%' or b.sex_upon_outcome like '%spayed%')`
			 - 괄호를 사용하여 우선 순위의 명확한 지정이 필요
				 - 괄호를 사용하지 않으면 where 조건과 and 조건에 일치하는 값과 or 조건에 일치하는 값을 전부 출력 
	 - 비교 연산 응용
		 - Not in
			 - A Table (Main), B Table (Sub) 의 Column 의 값을 비교하여 Sub 에 해당 되지 않는 Main 출력
				 - (A) Item_info Table	
					 - item_id 0, 1, 2, 3, 4	
				 - (B) Item_tree Table	
					 - parent_item_id 0, 1
				 - `select parent_item_id from item_tree where parent_item_id is not null group by parent_item_id` => 0, 1
				 - `select item_id, item_name, rarity from item_info where item_id not in (select parent_item_id from item_tree where parent_item_id is not null group by parent_item_id) order by item_id desc => 2, 3, 4` => Sub Query 비교

 - **논리 연산자**
	 - And
		 - Where 절로 지정된 조건에서 더욱 상세한 조건 지정을 위해 사용
	 - OR
		 - `Where (deptno1 = 10) or (deptno2 = 10) or (deptno3 = 10)`
			 - Column deptno1 ~ 3 의 값 중 10에 해당하는 값 출력
	 - Exists, not exists
		 - 데이터 요청 시 값이 조건을 만족하면 테이블 값 출력
		 - 서브 쿼리를 대상으로만 검색 가능
		 - 조건 검색 중 조건을 만족하는 경우를 1건이라도 발견하면 검색을 중지하고 출력
		 - `select 2 from dual where exists (select mem_name from member where mem_id=:x)`
			 -  Member table mem_id column 값이 존재하면 값 출력
			 - Not exists 는 반대로 값이 존재하지 않으면 값 출력
		 - https://gent.tistory.com/506
	 - Is Null , Is not Null
		 - Select 절에서 사용 시 참 또는 거짓 으로 표현
			 - 참 True / 거짓 False
			 - MySQL 의 경우 참 1 / 거짓 0 으로 출력
			 - Case When 문과 같이 사용하면 Null 여부 판별 가능
		 - Where 절, Having 절 에서 사용 시 참 인 부분만 출력

 - **집합 연산자**^union
	 - 출력되는 데이터 결과값을 연결한 상태로 출력
	 - Union
		 - 연결된 Table의 코드 중복을 제거하여 출력하는 합집합 (A + B)
	 - UNION ALL
		 - 연결된 Table의 코드 중복을 허용한 상태로 출력하는 합집합 (A + B)
	 - Intersect
		 - 연결된 Table의 공통된 부분만 출력하는 교집합 (A = B)
	 - EXCEPT
		 - A Table에는 존재하고 B Table에는 존재하지 않는 데이터를 조회 하여 출력하는 차집합 (A - B)

 - **Distinct**^distinct
	 - 중복 값을 제외하고 출력
	 - 검증을 위해 사용 & 성능 저하 요소
	 - `select distinct A Column from emp`
		 - A Column의 모든 중복 값을 제외 하고 출력
		 - `select distinct A Column ,B Column from emp`
			 - Distinct 뒤에 오는 모든 Column 은 Distinct 의 영향을 받음
			 - A Column, B Column 의 값이 동시에 일치하지 않으면 중복 아님
	 - 중복 값만 출력하려면 group by, having, count 사용
		 - `Select [column] from [table] group by [column] having count([column]) > 1`
		 - 두개 이상 column 값의 동시에 중복인 것을 출력하려면 group by에 사용되는 column 값을 두개 이상 사용
			 - `group by [column1], [column2] having count(*) > 1`
