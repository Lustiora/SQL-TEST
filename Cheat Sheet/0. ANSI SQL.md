| 순서  | SQL 절             | 역할                                                              |
| --- | ----------------- | --------------------------------------------------------------- |
| 1   | WITH              | 임시 결과 집합(CTE)을 정의합니다. 쿼리의 가독성과 재사용성을 높여줍니다.                 |
| 2   | FROM              | 쿼리의 대상이 되는 테이블이나 뷰를 지정하고 조인합니다.                                 |
| 3   | WHERE             | 개별 행을 대상으로 조건을 적용하여 필터링 합니다. (집계 함수 사용 불가)                   |
| 4   | GROUP BY          | 3번에서 필터링 된 행들을 지정된 열을 기준으로 그룹화 합니다.                         |
| 5   | HAVING            | 4번에서 그룹화된 결과에 조건을 적용하여 필터링 합니다. (집계 함수 사용 가능)               |
| 6   | SELECT            | 최종적으로 반환할 열을 선택하고, 계산식 및 집계 함수를 적용하여 값을 계산합니다.               |
| 7   | DISTINCT          | 6번에서 선택된 결과 행들 중 중복된 행을 제거합니다.                                  |
| 8   | UNION / UNION ALL | 둘 이상의 SELECT 문 결과를 결합합니다.(UNION 은 중복 제거, UNION ALL 은 중복 허용) |
| 9   | ORDER BY          | 직전까지의 최종 결과를 지정된 열을 기준으로 정렬합니다.                                 |
| 10  | LIMIT / OFFSET    | 정렬된 결과에서 반환할 행의 수를 제한하거나, 특정 위치부터 시작하도록 건너뜁니다. (페이징 처리)     |

- **서브 쿼리 (Sub Query)**
	- Limit 사용 불가
	- 복잡한 분석 및 집계상황 (대용량 DB) 에서는 Over 을 사용하는 것이 최적의 성능을 발휘하지만 특정된 단순한 목적을 출력하고자 하면 Sub Query 사용이 최적
	![[Pasted image 20251224114454.png]]
	- 스칼라 서브 쿼리 (Scalar SubQuery)
		- SELECT 절에 위치하며 한 레코드당 정확히 하나의 값을 반환하는 서브쿼리 (단일행, 단일칼럼을 반환)
	- 인라인 뷰 (Inline View)
		- FROM 절에 위치하는 서브쿼리로 결과는 반드시 하나의 테이블로 리턴되어야 한다 (별칭 부여 필수)
	- 중첩 서브 쿼리 (Nested SubQuery)
		- WHERE 절에 위치하며 결과집합을 한정하기 위한 서브쿼리  
		- 메인쿼리 컬럼을 참조할때 상관관계 있는 서브쿼리
