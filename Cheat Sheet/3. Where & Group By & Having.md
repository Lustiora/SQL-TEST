 - **Where 절**^where
	 - 검색에 사용되는 조건은 대소문자 구분 필요
	 - Select 절의 출력 값 조건 지정을 위해 사용
		 - `select empno from emp where deptno = 10`
			 - Emp Table 의 empno Column 중 deptno Column 에서 10인 값을 출력
	 - 다중 Table 사용시 일치되는 조건을 연결해줄 필요가 있음
		 - 연결되지 않을 경우 카테시안 곱 발생
		 - `where emp.deptno = dept.deptno`
			 - Emp Table 과 Dept Table 의 deptno Column 을 연결한 정보를 출력
		 - 조건 값으로 mem_id=:x 의 뒤에 붙는 :x는 동작 테스트시 값을 직접 입력하여 테스트 가능
	 - Year, Month, day 등을 사용하여 특정 날짜에 해당하는 부분만 출력가능
		 - `Year(Date Column) = 2021` => 2021년에 해당하는 부분만 출력
		 - `Month(Date Column) = 3 , Day(Date Column) = 1` ...
	 - 사용되는 구문
		 - 비교 연산자, 논리 연산자, 개별 행을 대상으로 하는 함수 (Round, To_char, …)

 - **Group By 절**^groupby
	 - Select 절에 사용되는 Column 과 동일하게 작성 해야 동작 (산술 연산자 제외)
		 - Select 절에서 별칭 (as) 으로 지정된 Column 이 있다면 별칭만 작성하여도 그룹화 가능
	 - Union 은 전체 Column 을 대상으로 한다면 Group by 는 특정 Column 을 기준으로 그룹화
	 - Union 등 사용시 가장 마지막에 사용
	 - Group by 적용된 Column 에 조건을 지정 한다면 Having 절을 사용
		 - `SELECT job, COUNT(*) cnt FROM emp WHERE deptno IN ('10', '20', '30') GROUP BY job HAVING COUNT(*) > 2`
			 - Emp Table 에서 job Column, Count 를 출력하되 deptno column 값이 10,20,30 인 것, Count 로 합계 집계 된 값 중 2를 초과하는 것을 출력
	 - 이름을 대상으로 Group By 사용시 동명이인 발생 여지 있음
		 - 이름 이외 ID 같은 고유식별자를 추가하여 방지

 - **Having 절**^having
	 - Group By 적용된 그룹에 대하여 조건을 지정하기 위해 사용
	 - Where 절과 동작 방식은 비슷하지만 그룹에 집계 함수 사용 가능
		 - `SELECT car_type, daily_fee from CAR_RENTAL_COMPANY_CAR group by car_type having sum(daily_fee) > 600000`
			 - 동작
		 - `SELECT car_type, daily_fee from CAR_RENTAL_COMPANY_CAR where sum(daily_fee) > 600000 group by car_type`
			 - 오류 / Where 절에 집계 함수 사용 불가
		 - `HAVING MAX(NAME = 'Milk') AND MAX(NAME = 'Yogurt')`
			 - Name Column 값이 'Milk' and 'Yogurt' 인 것만 표시
			 - `MAX(NAME = 'Milk')`
				 - 해당 부분이 참이라면 표시 1 출력 / 거짓이라면 미표시 출력 없음
